eInvoice matching tolerance

When the AMTEXP flag in the PO/AUTCHK parameter is set to Y the parameter PL/ETOLAMT gives a tolerance for the extended line amount when matching invoices to Purchase Orders and GRNs. The amount entered is in £’s so to include pennies you need to include a decimal point e.g. 1.50 incorporates a tolerance level of £1.50. The extended line amount will be considered matched if

(PO Amount – ETOLAMT) <= PL Amount <= (PO Amount + ETOLAMT)

When the PRCEXP flag in the PO/AUTCHK parameter is set to Y the parameter PL/ETOLPRC gives a tolerance for the line price. If the figure entered is a number then it is treated as a monetary amount, so in the UK 0.10 will be 10pence. The parameter may also be a percentage

Both monetary and percentage tolerances can be set in the parameter. Use a pipe separated pairing also showing the maximum PO line price for each tolerance eg in the UK 500,0.10|10000,0.10% would use a ten pence tolerance for prices up to £500, one-tenth of one percent for prices above £500 but below £10,000, and no tolerance would be used for prices £10,000 or above.

of the PO line price: 0.10% will be one-tenth of one percent of the PO line price. For a monetary amount the line price will be considered matched if

(PO Price – ETOLPRC) <= PL Price <= (PO Price + ETOLPRC)

For a percentage the line price will be considered matched if

(PO Price – (PO Price * ETOLPRC / 100)) <= PL Price <= (PO Price + (PO Price * ETOLPRC / 100))
