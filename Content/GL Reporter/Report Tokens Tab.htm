1.10 Report Tokens Tab

The Report Tokens tab enables filters to be created giving the user selections and filter to use when running the report. The Report Token tab enables you to define the filters to be used in the Report Rows or Report Columns. i.e. you might produce a single report that can be run for individual departments or sectors within the business (choosing which department or sector when you run the report).

A token is designated by a $ (dollar) sign followed a character to differentiate it. For example, $A = one token, $B = another token.

To use tokens effectively, you must embed them in your report CONSISTENTLY throughout the rows where (for example) makes an account selection.

For example, the account selection:

00.0000 to 00.9999

would just select accounts for department 00. But the selection:

$A.0000 to $A.9999

would allow the user to choose which department when they run the report. will print information regarding the selection automatically on the heading of a report containing tokens.

You could also use different tokens, to allow the user to select a range of departments; for example:

$A.0000 to $B.9999

Tokens can be embedded either in account rows or in columns within a report. Whichever option is used, they MUST then be used consistently throughout the report.

 

1.10.1 Adding a Token

The tokens can be used and selected from the type, which is either a filter used from the account record, or from the contract record.

You can see the Type dropdown list will display either the account details or the contract details.

The token is usually starts with a $ sign to show that this is a filter to be used.

Once you have decided what Tokens you want to have in the report, we can add the lines to the token tab.

 

Field:

Description:

Token:

The token name that is used in the report definition.

Description:

The description of the token that will be displayed as a run-time prompt when the report is run.

List:

The token can be used as a list of items.

Blank:

Leave blank, if a blank value is not allowed.

Type:

The type of value that should be assigned to the token at run-time.  This will depend where the token has been used in the report definition.

 

Tokens defined in Report Rows or Column Rows will be generated and displayed on the Report Tokens browse screen.

When you run the report you will be able to select the Sector or use a Lookup to select the correct option required.

 

 

1.10.2 Adding a Token in a Report Row

For example if you wanted to specify the Sectors for the report at run-time enter $ followed by a character e.g. S to represent sectors i.e. enter $S in the Sector field of an Account Line.

    Use the yellow Open button to amend the required.

    For example $S could be entered in Matches to enable Sectors to be entered at run-time.

 

1.10.3 Adding a Token in a Report Column

For example if you wanted to specify the Budget code for the report at run-time enter $ followed by a character e.g. B to represent sectors i.e. enter $B in the Sector field of an Account Line.

    Use the yellow Open button to amend the required line.

    For example $S could be entered into a Budget Line in the Report Column and a token entered in the Budget Field i.e. $B, to enable the budget code to be entered at run-time.

    Save the changes to the Report Column.

 

1.10.4 To Generate Report Tokens

    Select the Report Tokens Tab.

    If no report tokens have already been created a blank browse will displayed.

    To generate the Report Tokens configured in the report use the Generate Report Token function on the Options menu. 

Tokens defined in Report Rows or Column Rows will be generated and displayed on the Report Tokens browse screen.

    Use the yellow Open button on the browse screen to add a label for the Report Tokens, and optionally modify how they are presented.

Enter the required information for the Report Tokens.
