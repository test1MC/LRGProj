Parameters

The following new CB parameters control the functionality covered in this document.

Parameter

Description

QCRDFBNK

If this is set to Yes, the Bank Code on a Quick Cash Receipt batch defaults from the bank set up in Configuration.

QCRHIJOB

If this is set to Yes, the number cannot be entered on the Quick Cash entry screen, and the (number) and Name fields are hidden. This means that the receipt cannot be allocated to invoices and certificates by ; therefore the user will need to enter a customer code, meaning the receipt can only be allocated to invoices and certificates for that customer.

PARTALOC

 If this is set to Yes, when an Allocate Customer Cash batch with unallocated cash is posted, a customer cash line for the unallocated amount is created automatically.

LVCCPACB

If this is set to Yes, in Allocate Customer Cash, it is not possible to enter or analysis line and partially allocate the amount entered, with the intention that the remaining amount posts to the or Ledger as unallocated cash. Hence if or analysis line is entered it must be fully allocated. This is to ensure that if a customer cash record is selected but not fully allocated, then the unallocated part is left in instead of being posted to the or Ledger.
