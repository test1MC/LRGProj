Quick Cash Receipt

The following changes have been made to Quick Cash Receipt Entry.

Default Bank on Batch Header

If the new CB parameter QCRDFBNK is set to Yes, the Bank Code on the batch header screen defaults from the Default Bank set up in Configuration.

User access to the Bank Code field can then be restricted to Read Only using Field Access Maintenance.

Entry Screen

The Customer Name field has been moved from the Body Update form to the Body form.

If there is any existing unallocated cash for a customer, this is displayed after the Customer Name field, with the label Prior Unallocated Cash.

If the new CB parameter QCRHIJOB is set to Yes, the code and name fields are not displayed.
