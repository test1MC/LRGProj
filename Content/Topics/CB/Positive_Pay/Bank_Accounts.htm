Positive Pay – Bank Accounts

Bank account setup has been changed to add extra fields related to positive pay. These fields are only displayed if the CB parameter USEPOSPY is set to Yes.

Field
Description

Export Checks

Whether posted should be sent to log. Default False

Export Unposted Payroll

Indicates what should be created on the log.

None – nothing

Printed – Printed

Manual – Manual

Override Account Number

Allows Pos Pay to report different account number than is printed on .

Control Record

Specifies Control Record should be exported as line 1 of export

Format

The positive pay format to use.

Default File Name

Encoded value of positive pay file name to use.

If no file name Is specified in bank setup then a default name such as “pprpt-xx-yymmdd-nn.txt” will be created where xx = bank id and yymmdd = today’s date and nn = sequence number for today. A file name may be specified by bank with the following components:

·         Text

·         %d% - includes date in this position using mask .. ex %dMMDDCCYY%

·         %s% - sequence number using mask .. ex %99%

·         %b% - bankid

Example File Name – mypospay-pnc-%dMMDDYY%%s99%.txt

Additional Format

List of available positive pay formats (some banks require summary file)
