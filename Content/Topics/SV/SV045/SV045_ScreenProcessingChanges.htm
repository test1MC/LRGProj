Status Maintenance – Zero Margin box

Register

A new Zero Margin option is processed on a (or single Item) Status change, as specified in Section 1.3.

NOTE:
The Zero Margin option is connected to Status change processing only; it is not used to automatically keep a Zero Margin when you update Items/Resources. Any user input made in Pricing (Total, Simple, or Detailed) results in changes in Summary Pricing.

Processing

Processing that occurs when running the Update Status action is as follows:

  • If launched for a belonging to / , the Zero Margin flag is ed on the selected status.
  • If the Zero Margin flag is set, values are set on the Items (resources).
  • The processing of values depends upon configuration and the Pricing Method:
  • Detailed : Each Item of the is processed separately, and the resources update the Item summary and the Summary Pricing values.
  • Total and Simple Priced : Same processing occurs for a single Item that forms the .
  • If parameter SV/VOHDRREV = Y, the -only resource (not displayed but used in the Item) is processed as part of the Item.

  2. System will determine the possibility to apply Zero Margin and assign values as specified below.

  2. SumTkupint > SUM(Estimated ): Zero Margin is not possible to achieve, set 100% on all resources (ext.takenup = Estimated ). In this case show appropriate warning message.
