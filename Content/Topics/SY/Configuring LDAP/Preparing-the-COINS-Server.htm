Preparing the Server

The first task is to load the requisite software on the server.

LDAP Directory user authentication utilises open source OpenLDAP and DNS libraries to communicate with Corporate Directory servers on the local area network.

 

Do not proceed with this or any further tasks if you have not been able to successfully identify your network’s LDAP directory servers, as outlined in Confirming the Active Directory server resolution.

 

Perform the following steps using the administration account on your server. You must be logged into the server using the server’s root account.

  1. Log into the server using the administrative account login details.
  2. Determine whether or not the OpenLDAP package is already installed on the server:

If OpenLDAP is not installed then proceed to step 3.

If it is already installed you can proceed to step 4.

  1. Install the OpenLDAP libraries.
  • From the command prompt issue the command: yum install openldap
    The following will be displayed:

  • Choose y and press enter to continue.

  • If no errors are encountered you will see the line: Complete!
  1. LDAP Directory user authentication requires the software to be able to access the OpenLDAP libraries from a platform independent area. software utilises the /usr/local/lib file system path as a common repository for operating system shared libraries. To allow the software to use the OpenLDAP and DNS shared libraries, you will need to link these libraries into the common repository:

 

The name (libldap-2.4.so.2 & libresolv.so.2) and location (/lib64) of the shared libraries may vary depending on the version of the library and whether you are running a 32bit or 64bit operation system.

 

You have now successfully installed the required open source software on your server and made the required shared libraries available to the software.
