Testing the installed certificates

To avoid any potential issues with user authentication, we recommend that the installed Certificate Authority certificates are tested before use. You can then be confident that any connectivity issues with the network LDAP directory servers are not due to certificate issues.

Before proceeding with this sub-task, you will need to know the host name or IP address of each of the LDAP directory servers. The test requires connecting to the server and verifying the certificate use.

Perform the following steps using the administration account on your server. You must be logged into the server using the server’s root account.

 

  1. Log into the server using the administrative account login details.
  2. Navigate to the created certificate repository. In this How To guide we are using: /usr/local/etc/coins/certs

  1. For all of the installed certificates, you will need to attempt a connection to the associated LDAP directory server as shown below. Replace the <LDAP Server> token (in the command below) with the host name or IP address of the LDAP directory server. Also note that the highlighted certificate file name (the argument to the –CAfile parameter) needs to be the same as that created from the associated LDAP server’s Certificate Authority’s certificate.

If the secure sockets layer connection was successful, the output of the above command will show a Verify return code of zero. If the installed certificate cannot be used to verify the secure sockets layer connections, then you will see a Verify return code of non-zero, for example:

 

Verify return code: 21 (unable to verify the first certificate)

 

If there are any failures in verifying the usability of a particular certificate file, then will not be able to utilise secure sockets layer when communicating with the respective LDAP directory server. You will need to contact the LDAP directory server’s administrator to ensure that the correct Certificate Authority’s certificate has been extracted for that particular LDAP directory server.
