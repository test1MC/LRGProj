Confirming the Active Directory Server Resolution

The LDAP Directory user authentication, implemented in , uses your network’s domain name lookup (DNS) services to identify all available directory servers that are available for user authentication.

The server must be able to resolve directory servers using: _ldap._tcp.<domain> where <domain> is the domain name of your local area network. The following task shows the steps involved in testing the ability of your server to identify available LDAP directory servers.

No LDAP directory servers

If your server is unable to identify any LDAP directory servers on your local area network then you have two options:

  • Talk to your local area network administrator to determine whether or not your server can be configured to access a domain name server that is able to identify LDAP directory servers on your local area network.
  • Install and configure DNS server software on your server. You can choose either a fully featured DNS server, for example BIND9, or a light weight DNS forwarder like Dnsmasq. This will allow the server to be configured in such a manner that LDAP directory server identification can be implemented on the server.

 

Both of the above options are outside of the scope of this How To guide.
Please refer this to your server and/or local area network administrator for resolution.
