LDAP Configuring User Accounts

To be able to associate a user account with an account within your Corporate Directory, you must map the user accounts to a user principal with the corporate LDAP directory.

For Microsoft Active Directory, you can use <userid>@<domain> for example: johdoe@example.local

 

Do not proceed with this or any further tasks if you have not been able to successfully complete the steps outlined in Preparing the server. Once a user’s account has been configured to use LDAP Directory user authentication, the affected user will not be able to log into unless can verify the entered credentials against your corporate LDAP servers.

 

Complete this task using OA.

  1. Log in to OA using an administrative account that has the ability to alter system parameters.
  2. Navigate to System module > System Setup > SY Parameters.
  3. Find the LDAPON parameter and update it to ‘Y’ as shown below:

Once this parameter has been set to ‘Y’, LDAP Directory user authentication will be enabled for user accounts.

  1. Navigate to System module > User Maintenance > Users.
  2. Find a user account that you would like to change to LDAP Directory user authentication and open the user record, as shown below:

  1. Go to the Detail tab on the user record

  1. Enter the user’s LDAP Directory user principal identifier in the LDAP User@Domain field . Once this field contains a value, the password, password expiry and Domain\User fields will be hidden.

  2. Save the update and verify that the modified user can use their network login password to access . Once this has been verified, proceed to find and update all other user accounts that you would like to use LDAP Directory user authentication against.

A mixture of LDAP Directory user authentication and passwords can be employed for your user base. Simply leave the LDAP User@Domain field empty on accounts where the password token is required. Note however that a mixture of both authentication mechanisms cannot be combined on a single account.
