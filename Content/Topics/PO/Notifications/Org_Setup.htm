eCommerce Organation Setup

To allow PO notifications to be configured for individual or subcontractors, a new PONOTIFY record has been added to Organation Setup. The settings in this record override the global ones at the , subcontractor or office level.

Organation Setup Screen

The following settings are available:

  • <EnableNotifications>[Y/N]</EnableNotifications> – enables or disables notifications for the specific company and office. If this is not set, then the value of the PO/DEFNTFEX parameter is used.
  • <MailTo>[valid email]</MailTo> – overrides the office email address for notifications.
  • <EmailTemplate>[valid PONOTIFY-EXT template]</EmailTemplate> – specifies a custom email template to use for notifications.
  • <AllowUncommit>[Y/N]<AllowUncommit> – allows orders to be uncommited after an email has been sent to the or subcontractor; default is N.
  • <SendPOTypes>[pot_types list]</SendPOTypes> – a can-do list of Order Types to send notifications for; default is ‘*’
