Integration Application Settings

The integration application needs information about your environments. Use the Edit the Environments screen during the installation to provide this information.

There are three partially-configured environments: LEARN, CPLEARN and LIVE. To edit the settings for one of these, first select the environment (at the top of the screen) then fill in the fields (see the table below).

You can edit settings for additional environments; either click New to create a blank environment, or select an existing one and click Copy.

The fields and their meaning are as follows:

Parameter

Description

AppServer

The Appserver connection string as advised by the system administrator. Mandatory.

For example: AppServer://coins.server .internal:5192/asappqafb

Logging Level

Logging level.  1 – basic, 2 – intermediate, 3 – verbose, 4 – debug.

Exchange Web Service URL

The URL for the Exchange Web Services application as advised by the Exchange administrator. For example: https://exchange.server.local/ews /Exchange.asmx

Either Exchange Web Service URL or Email for Auto Discover must be specified.

Email for Auto Discover

The Email address to use for auto-discovery of the Exchange Web Service server.  If specified then this will be used instead of the Exchange Web Service URL.

Active Directory Account

The Active Directory replication account (see Active Directory Replication Account).  Useful for debugging.  If omitted then the login credentials are taken from the logged in user.  Optional.

Password

The password for the user specified in the User field.  Optional.

NOTE

The configuration file created by the installation stores the password as text, so you should take care regarding who can access the file.

Domain

The domain for the user specified in the User field.  Optional.

User

A valid user in the environment used for recording record additions, deletions etc.  Mandatory.

Enable replication of Contacts

Whether to enable replication of Contacts. Optional.

Relative Path

The relative path to the public folder used to hold contacts. This is required if Enable replication of Contacts is ed.

Workflow Emails

Whether to enable import of Workflow emails from a public folder or mailbox. Optional. If ed, the Relative Path or Workflow Mailbox must also be set.

Relative Path

The relative path to the public folder used to hold workflow emails for import. Ignored if Workflow Mailbox is set.

Workflow Mailbox

The mailbox name for inbound workflow emails.

Project Emails

Whether to enable import of project emails from public folders. Optional.

Relative Path (inbound)

The relative path to the public folder which is the parent folder of the individual project folders for import.  Required if Project Emails is ed.

Relative Path (archive)

The relative path for archiving of imported messages. Optional.

If this is populated, an archive tree of folders is set up to mirror the import tree of folders, with corresponding folders for each project and messages will be moved from the project folder to the archive folder after importing. If this is blank, messages are deleted after importing.

Enable Activity Workbench Replication

Whether to enable replication of Activity Workbench tasks and appointments. Optional.

Replication Code

The code for the replication process. This can be any value, but is compared to the Replication Mail Server field on the user record. Only users with the same value in the Replication Mail Server field will be replicated. Required if Enable Activity Workbench Replication is set to Y.

Monitor Folders

Whether to enable processing of monitored folders. If ed, all monitored folders defined in OA will be checked for inbound messages and these will be passed to OA for processing.

Process Mail Assistant Sent Emails

Whether to enable the loading into of sent emails that have been attached to a record via the Mail Assistant.

When you have filled in all the details, click Next.

If you are installing the application to run as a console program, the next screen allows you to set up a scheduled task; see 1, Console Schedule Task Setup. Otherwise, the next screen allows you to confirm that you want to install the application.

When you click Install, the installer creates the necessary files in the location you specified.
