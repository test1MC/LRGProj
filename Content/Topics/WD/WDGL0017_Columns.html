GL Reporter - Columns

 

The columns control what information is shown on each line.  You can select from a number of different data items, and specify which row types each one is shown on.

When adding a column, first select the type of item you want the column to contain. 

 

The Label is generally for your reference when configuring the report, to explain what the column shows; however, for Literal columns it is the text to print on the report.

The Show On field determines which line types the column will be printed on.  Use a comma-separated list of one or more from the following:

B Blank rows
Dn Account rows
Fn Sort footers
Hn Sort headers
Jn rows
Sn Standard text rows
X Additional text rows

Where n is a number that represents a level.  For account and rows, you can think of levels as variations of the same type of line on which some of the columns may be different.  For example, if you want one set of account lines to show closing balance and movement, and a second set to show budget and forecast, give the two row types different levels, and put the relevant level number after the "show on" code for each column.

Item type

Show On
Account code D.
Account description D.
Closing balance D1
Movement D1
Budget D2
Forecast D2

For header and footer lines, the levels correspond to the sort levels as defined on the row.  A instead of a number means "show on all levels for this type of line". 

Column widths

In the Col field, specify the starting position for the column.  The starting position is (approximately) the number of characters across the page.  Note that if the format of a column is wider than the difference between the two Col fields, the following columns will not be aligned.

Column Headings
