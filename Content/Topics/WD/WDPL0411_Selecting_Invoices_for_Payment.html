Selecting Invoices for Payment

allows you to select outstanding transactions for a . The amounts will then be automatically calculated to create a payment ( or ) for the total amount. Credit transactions (indicated by a negative amount) can also be selected, which will adjust the payment total accordingly.

There are two procedures available that allow you to select invoices for payment.

  • Select/Deselect Invoices (Manual) - allows you to create payments by manually selecting outstanding invoices for one at a time.
  • Select/Deselect Invoices (Batch) - automatically selects (or deselects) outstanding transactions for one or more , according to the selection criteria you specify.  For example, you can select invoices by due date.  Depending on how has been configured, you may or may not be able to use Select/Deselect Invoices (Batch) to select held or registered invoices, or those where unallocated payments exist. 

Because both the procedures interact, you can use either method of selection, or you can combine manual and batch selection: for example, you can select a group of invoices using batch selection, and then deselect individual invoices manually to stop them being paid.

Regardless of which selection procedure you use, one payment payment will be created per , to create a payment batch.

 

Generate Payment Selection

Select Invoices for Payment

From the Menu select Payments then Select/ Deselect (Batch)

Choose the Currency that the payment will made in and in the Select/Deselect field, choose Select

If you wish to run a report only (for example, just to get an indication of the payments that may be included in a run) without actually creating a Payment Batch, set the Select/Deselect field to Draft.

If you want to produce a section in your report that will show invoices that have not been selected, and the reasons why, then you need to check the "Show Unselected" field.  Similarly, you can choose to show all invoices where insurances warnings exist.

Main Tab

On the Main tab select the criteria that you want to use to batch your invoices for payment

Field

Description

Selection Date

The date on which selection of invoices is based (depending on the value in the Select By field).

Discount Date

The date on which the taking of discount is based. If this date is within the discount date on the invoice, then discount will be taken.

Select By

Select whether Invoices with a due date, invoice date or discount date on or before the Selection date will be selected for payment

Include non-contract invoices

Whether to include non-contract items in the list - for example invoices which have an overhead department assigned.


It is possible to print cheques only or payments or both. Make this selection by checking the appropriate fields.

There are a variety of criteria that can be used to batch the invoices. This can be done by selecting the relevant tab and completing the necessary fields.

Supplier Selection

You can enter a range of supplier codes e.g. from EBL001 – HEYR001. will then batch any invoices due for payment to suppliers within this range. You can also use a range of Payment Frequency or Group Analysis. This would have been set up in the Supplier Account Details.

Selection

It is possible to use details of the contract to select invoices for payment. For Example. HH8016 - HH8018 this will give all outstanding invoices for any contracts that fall within this range
