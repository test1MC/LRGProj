Setting Up Function Access Using Main Menu Security

Using Main Menu Security, you can set the function access permissions for a group or a user. 

  • If you are using Menu Item Security you can set the access permissions at the level of menu items; the child functions inherit the permissions from the menu item.
  • If you are not using Menu Item Security, you can set the access permissions for the menu items, and follow a link to a screen that lets you set the permissions for the child functions of each menu item. 

How To Set Up Menu Item Security Using Main Menu Security:

    1. Go to Main Menu Security.
    2. Enter the user, group or sub group for which you want to specify access.
    3. Enter the code for the top-most menu for which you want to specify access.

      The Main Menu Security screen shows all menus and menu items from the start menu that have themselves as a parent (that is, they are considered as main menu items). 

      You can specify the following access to functions:

Not Using Menu Item Security 

Using Menu Item Security 
  • No. The User or Group is not allowed to access the function.
  • Yes.  The User or Group is allowed access to the function. 
  • Yes.  The User or Group is allowed Read Only access to the function.  Only those child functions with an access type of Read Only will be available; functions with an access type of Update, such as the update buttons, will not be available.
  • Yes - Update.  The User or Group is allowed Update access to the function.  Child functions with Read Only and Update access types will all be available.
  • Group.  The access to the function is taken from the group; each user or group has  one or more other groups assigned to it; looks at each group in turn until it finds an explicit setting (No/Yes/Yes-Update).  If none is found, access is denied.
  1. To update access for several functions at the same time, select one or more records and either:
    • Use the Choose Action menu or multiple update button to set the same access for all the selected menu items.
    • Use the concurrent update button to set the access for each item separately.
  2. To update access for a single function, use the button.
  3. You can set the access for child functions by clicking the link in the Child column, and using the Choose Action to set the permissions for individual child functions.

    If you are using Menu Item Security, by default, the access to any child functions will be the same as the access to the parent function; setting them on this screen will override this default setting.

To use Menu Item Security, the SY parameter MENUSEC must be set to Y.
