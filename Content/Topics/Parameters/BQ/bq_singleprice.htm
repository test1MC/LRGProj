BQ/SINGLEPRICE - Only one Price allowed with the same Resource Supplier and Development

Whether to accept multiple Price Workbench records for the same resource// combination and overlapping valid periods.

Y = You are warned if you are adding/updating the price with the same resource// and overlapping valid periods. The "To Date" for any existing prices with the same combination will be updated to ("From Date" - 1), thereby allowing a pricing history to be built up and the programming of future price changes.

N = Multiple valid prices are allowed to be held with the same resource// combination and valid periods.
