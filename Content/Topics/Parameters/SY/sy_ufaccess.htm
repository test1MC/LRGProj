SY/UFACCESS - User function access date/time

The date and time stamp for compiled user security. This can speed up access to functions because only needs to consult one table to evaluate whether a user has access.

To enable compiled function security, enter an exclamation mark (!) as the value of this parameter.

When the next user logs in, stamps the parameter with the date and time. From then on, the first time a user accesses a function, checks the access permissions for the user, and records whether the user has permission to access the function. Subsequent access requests are quicker because can consult just one table to evaluate whether a user has access.

If user groups, roles, or group security are changed after compiled function security has been enabled, you must re-set the parameter to ! for the updated security settings to take effect. This has the effect of making all the previously-saved security settings invalid; then re-evaluates and saves permissions (as when this feature is originally switched on).

If the parameter is left blank - or restored to blank - then this functionality is not applied and evaluates function access every time.

See also: Compiling Function Security.
