SC/ALTDATES - Allow draft certificates alternative date positioning and validation

Whether to allow control of the sequence and validation of date fields on the Draft Certificate header (Y or N).

Y = The Claim Date field is labelled 'Input Date'. This is the date the subcontractor's application/valuation is entered into COINS by the user. This date defaults to the current date.

The Valuation Date field is labelled 'Works Valued Up To'.

The Certificate Date field appears below the 'Works Valued Up To' field.

The Reference field defaults to the user's User ID.

N = The fields are displayed in the order Claim Date, Certificate Date, Valuation Date.

The Reference field can be any text.
