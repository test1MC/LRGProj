SC/INSMAP - Mapping for Extended Insurance

When converting from old insurance to configurable insurance, this allows you to associate configurable insurance types with the four old insurance types, so you don't lose any data previously held. The old method is still available in and the four insurance types are kept in sync, so any reports that use the old method and have not changed will still work.

This is a pipe-separated list of the names of configurable insurance types that correspond to the four old insurance types.

Entry 1 = Public Liability;
Entry 2 = Employer Liability;
Entry 3 = Professional Indemnity;
Entry 4 = Contractors All Risk.

When you set up or update a configurable insurance type that has the same name as one of these entries, COINS loads policy data (insurer, policy number, limit amount and date) from the old insurance, and updates the old insurance automatically if the configurable insurance type changes.
