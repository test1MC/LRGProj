Running Report Pack

You can run report packs using the Run button on the Report Pack maintenance screen, or the Report Pack Reports maintenance screen. 

This launches a desktop frame with the report selection.

Field Definitions

Report Pack

The report pack to be run.  The current pack is preselected but it is possible to select a different report pack.

Only report packs that are relevant to the current company are shown.

Run Date

The date for the offset calculations.  The dates for each of the saved or scheduled reports in the pack are calculated based on this date.  Typically you would enter the end of the period and it might calculate the range of dates that are the start and end of that period for the reports to run.

This causes each of the reports in the report pack to be run (one at a time and in the sequence defined).

All reports are generated as individual reports, but when all reports are complete those that are marked as output = Report Pack are consolidated in the PDF and Excel output for the report pack (the first entry on the queue).  Those that are defined as output = Separate are excluded from the consolidated PDF and Excel.

Each consolidated report in the Excel output is placed on a separate tab.

If you email the report pack, reports marked as Separate are sent as separate emails, and those marked as Report Pack are sent as a combined report of all the grouped reports.
