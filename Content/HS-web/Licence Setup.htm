Licence Setup

 

First you will need to setup a user id that is setup for web services.  You can only do this through OA User Maintenance.  You need to ensure that the ‘Webservices’ box is ticked.  See below:

 

Note the number of users that can be setup for web services is controlled by the licence for module code NWS. 

You will also need to ensure that you have the required licences for the House Sales Connection Enquiry (NHSE) and optionally the House Sales Connection Update (NHSU).  See below:

 

 

The licences for the modules NHSE and NHSU control the maximum number of developments that can made available through the HS connection interface.  Note on some older patches of you may need to add these modules into the list.

 

You will also need to obtain a licence to be able to run the individual web service objects, this should be entered on the ‘Extras’ tab in module licence maintenance.  See below:

 

 

Under the X-WSERVICE extra you need to specify a list of the soap objects you are licenced for.  To run all the HS connection objects you can specify a can-do list e.g. hsws*

Note this ‘Extras’ screen forms part of the licence control cross check and you should ensure that you obtain the correct licence certificate with the soap objects you want to run from before amending this screen.

 

Once the licences have been obtained, you will then need to indicate which developments are available for enquiry and update access.  A new maintenance browse ‘Connection Licences’ in OA has been produced for this.  This can be fond under the OA House Sales/Plot Sales menu.  See below:

 

 

Note on patches prior to 0609 this will not be on a menu.  You will need to add the function %WVP2111BVSI to an OA menu.
